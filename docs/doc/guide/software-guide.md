# 权限与解锁

SecScore 通过“权限状态 + 密码”来保护关键操作，避免误操作或他人随意修改数据。

## 权限模型

- 管理权限：学生管理、理由管理、结算、数据管理等全部功能
- 积分权限：仅允许积分相关操作（加分/扣分、撤销等）
- 只读：仅查看排行榜与历史，不允许修改

## 解锁方式

在界面右上角查看当前权限状态：

- 若显示“只读”或“积分”，点击“输入密码”解锁到更高权限
- 若需要进行学生管理、结算或数据导入，请确保已解锁到“管理权限”

## 密码建议

- 管理密码用于保护“结构性操作”（名单、理由、结算、导入导出）
- 积分密码用于保护“日常记分操作”，便于多人协作时控制风险

## 找回字符串

如果你开启了密码保护，建议妥善保存应用提供的“找回字符串/恢复信息”。当忘记密码时，可按应用内提示使用该信息进行恢复。

## 首次使用推荐顺序

1. 设置管理密码与积分密码（如需要）
2. 导入学生名单并检查姓名列是否正确
3. 维护常用“理由预设”，减少重复输入
4. 开始日常加/扣分并按需撤销
5. 每阶段结算归档，定期导出 JSON 备份
