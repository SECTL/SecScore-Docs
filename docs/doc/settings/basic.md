# 基本设置

本页介绍 SecScore 的常用设置项与推荐配置，帮助你在不同教学场景下稳定、可控地使用积分系统。

## 主题与外观

- 主题切换：支持浅色/深色主题切换
- 主题编辑器：可视化调整主题相关配置（如 CSS 变量与配色），并保存为自定义主题
- 界面缩放：调整窗口与字体的整体缩放比例，适配不同分辨率与投影环境
- 如界面元素显示异常，优先尝试切换主题或重启应用后再检查

## 权限与密码

SecScore 提供三种权限状态：

- 管理权限：学生管理、理由管理、结算、数据管理等全部功能
- 积分权限：仅允许积分相关操作
- 只读：仅查看，不允许修改

常见设置建议：

- 为“管理权限”设置强密码，避免误操作或他人修改数据
- 如需要多人共同操作，可为“积分权限”设置单独密码，降低风险
- 忘记密码时依赖“找回字符串”，建议妥善保存：见 [权限与解锁](../guide/software-guide.md)

## 数据管理（导入/导出）

- 导出数据：建议在大改动前先导出一次 JSON 作为备份
- 导入数据：用于跨设备迁移或恢复备份，导入前建议先保留当前数据副本
- 导入会覆盖数据，建议先做一次导出备份：详见 [数据导入导出](./other.md)

## 日志

- 查看日志：用于定位“无法启动/闪退/操作未生效”等问题
- 导出日志：反馈问题时附带日志与版本信息更容易定位
- 清空日志：在问题解决后可按需清理

## 托盘与关闭行为

- 关闭主窗口通常不会退出应用，而是隐藏到系统托盘
- 托盘菜单支持“显示主窗口 / 退出应用”，双击托盘图标也可唤起主窗口
- 可按需要调整“关闭行为/最小化到托盘”等相关选项（以应用版本实际提供为准）

## URL 协议（secscore://）

安装版应用可注册 `secscore://` 协议，用于从外部快速打开指定页面。

- 例如：

  ```text
  secscore://settings
  secscore://leaderboard
  ```

- 支持的命令与行为说明见 [自定义 URL 协议](./ipc_url.md)

## 推荐工作流

1. 首次使用先设置密码与权限
2. 导入学生名单并完善分组信息
3. 维护常用“理由预设”，减少重复输入
4. 每周或每阶段定期导出 JSON 备份
