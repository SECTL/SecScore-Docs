> 💾 **备份、迁移与恢复数据**
>
> 数据导入导出用于备份与跨设备迁移。日常强烈建议定期导出一次 JSON 备份，避免误操作或设备损坏导致数据丢失。

## 📤 导出 JSON（备份）

- 建议在以下场景导出一次备份：
  - 大规模记分前后
  - 调整大量理由预设
  - 进行结算前
  - 更换电脑/重装系统前
- 导出文件可用于恢复到导出时的状态

## 📥 导入 JSON（恢复/迁移）

导入用于恢复备份或跨设备迁移，但需要注意：

- 导入会覆盖现有数据（学生、理由、积分记录、结算历史等）
- 导入前建议先导出当前数据，避免覆盖后无法回退
- 安全相关设置（例如密码）通常不会随导入写入（以应用版本实际行为为准）

## 🧾 结算与数据关系

- 结算用于归档一个阶段并清零当前积分：见 [结算与历史](./history.md)
- 结算前建议先导出一次 JSON，便于误操作后回滚

## ✅ 建议工作流

1. 每周或每阶段结束导出一次 JSON
2. 结算前导出一次 JSON，再进行结算归档
3. 需要恢复时，先确认要覆盖的设备/班级数据无误，再导入
