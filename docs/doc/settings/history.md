> 🧾 **结算与归档**
>
> 结算用于把一段时间内的积分记录“封存成一个阶段”，并开始新的计分周期。

## 结算是什么

你可以把“结算”理解为一次阶段性结账：

- 将当前阶段的积分记录归档为一个历史阶段
- 生成该阶段的排行榜，供后续回看
- 将所有学生当前积分清零，开始新的阶段计分
- 学生名单与理由预设保持不变

## 什么时候需要结算

常见场景：

- 每周/每月结一次，形成周期性统计
- 阶段考试/活动结束后结算归档
- 需要重新开始计分，但又想保留上一阶段排行

## 结算前建议

- 先确认权限为“管理权限”（结算通常属于高权限操作）
- 建议先导出一次 JSON 备份，避免误操作后无法恢复：见 [数据导入导出](./other.md)

## 结算会影响哪些数据

- 会变化：
  - 当前积分（清零）
  - 当前阶段的记录会被标记为已归档，不再参与“当前阶段”的统计
- 不会变化：
  - 学生名单
  - 理由预设
  - 历史阶段的排行榜与记录

## 结算历史怎么用

结算完成后，你可以在“结算历史”中：

- 查看每个阶段的排行榜
- 对比不同阶段的表现变化
- 回溯某个阶段结束时的积分状态

## 常见问题

### 结算后还能恢复到结算前吗

如果你需要恢复，优先使用结算前导出的 JSON 备份进行导入恢复：见 [数据导入导出](./other.md)。
