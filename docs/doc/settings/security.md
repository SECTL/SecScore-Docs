> 🔒 **用密码与权限保护关键操作**
>
> SecScore 通过“权限状态 + 密码”保护学生管理、理由管理、结算与数据导入导出等关键操作，避免误操作或他人随意修改数据。

## 🧭 权限与密码

SecScore 常见权限状态：

- 管理权限：全功能（学生管理、理由管理、结算、数据管理等）
- 积分权限：仅允许积分相关操作（加分/扣分、撤销等）
- 只读：仅查看排行榜与历史，不允许修改

密码通常分为两类（以应用版本实际提供为准）：

- 管理密码：解锁到“管理权限”
- 积分密码：解锁到“积分权限”

## 🔓 解锁与锁定

- 右上角会显示当前权限状态
- 若显示“只读/积分”，点击“输入密码”可解锁到更高权限
- 点击“锁定”可切回只读状态
- 未启用密码时通常默认视为管理权限

## 🧩 找回字符串

启用密码后，应用会提供“找回字符串/恢复信息”用于忘记密码时恢复访问：

- 建议在首次设置密码后立刻保存到安全位置
- 找回字符串一旦丢失，忘记密码会非常麻烦

## ✅ 推荐设置

- 管理密码设置强一些，用于保护结构性操作
- 多人协作记分时，给“积分权限”单独设置密码降低风险
- 进行导入/结算前先导出一次 JSON 备份：见 [数据导入导出](./other.md)

更多解锁说明见 [权限与解锁](../guide/software-guide.md)。
