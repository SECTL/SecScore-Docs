# Development & Contributing

Contributions to SecScore are welcome: bug reports, documentation improvements, code changes, translations, and more.

## Reporting Issues

- GitHub Issues: https://github.com/SECTL/SecScore/issues
- Please include: OS, SecScore version, repro steps, screenshots/logs, expected vs actual behavior

## Code Contributions

1. Fork the repo: https://github.com/SECTL/SecScore
2. Create a branch and implement your change
3. Open a PR and describe intent and impact

## Docs Contributions

This repository is the SecScore documentation site. You can edit the corresponding page and submit a PR.
